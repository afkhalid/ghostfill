# Security Best Practices Report

## Executive summary
GhostFill now aligns much better with frontend security best practices: the shipped browser package no longer stores provider API keys, no longer sends existing field values or scraped page context to AI providers, and exposes a backend-only AI contract instead. I did not find any remaining code-level secret exposure in the package itself. The residual risk is operational: consumers still need to secure any `/api/ghostfill` backend they add, and preset prompts remain same-origin localStorage data that should stay non-secret.

## Critical

No critical findings remain in the shipped package after the refactor.

## High

### SBP-001: Optional backend AI route still needs consumer-side auth, rate limits, and body-size controls
- Severity: High
- Impact: A copied `/api/ghostfill` route can be abused for provider spend, quota exhaustion, or backend load if it is exposed without auth or limits.
- Evidence:
  - The browser package posts AI requests to `ai.endpoint` or `/api/ghostfill` (`src/ai.ts:139-172`).
  - The README example route validates provider/shape but intentionally leaves auth and rate limiting to the integrator (`README.md:109-132`).
- Fix status: Documented and partially mitigated.
- Recommended follow-up:
  - Require authenticated access to the route in real deployments.
  - Add per-user/IP rate limits and request-size caps.
  - Keep the route dev-only unless there is a strong product need.

## Medium

### SBP-002: Preset prompts are still stored in localStorage and readable by same-origin scripts
- Severity: Medium
- Impact: If a host application suffers same-origin XSS, any GhostFill presets saved by the user can be read and exfiltrated.
- Evidence:
  - Settings and presets are persisted in localStorage (`src/overlay.ts:53-80`, `src/overlay.ts:1328-1338`).
  - The README now warns that presets should remain non-secret (`README.md:138-143`).
- Fix status: Accepted residual risk.
- Recommended follow-up:
  - Keep presets non-secret.
  - Add a future memory-only or session-only preset mode if you need stronger local privacy.

## Low

### SBP-003: Deprecated `apiKey` remains in the public type surface for backwards compatibility
- Severity: Low
- Impact: The runtime ignores it, but the leftover field can still mislead consumers into thinking client-side keys are supported.
- Evidence:
  - `GhostFillOptions` still exposes a deprecated `apiKey` field with a warning comment (`src/types.ts:2-13`).
  - The runtime warns and ignores it (`src/overlay.ts:494-497`).
- Fix status: Mitigated in runtime, not yet removed from API shape.
- Recommended follow-up:
  - Remove `apiKey` in the next major release.
  - Keep all docs and examples backend-only.

### SBP-004: Source maps are generated by default for package builds
- Severity: Low
- Impact: There is no secret in the bundle today, but publicly distributed maps still reveal implementation detail that some consumers may prefer not to publish.
- Evidence:
  - `tsup` builds with `sourcemap: true` (`tsup.config.ts:3-9`).
- Fix status: Informational.
- Recommended follow-up:
  - Leave as-is for developer ergonomics, or document publishing guidance for consumers who redistribute the bundle.

## Addressed findings

### Fixed-001: Browser-side provider key storage and collection removed
- Previous risk: Provider keys were stored in localStorage and sent directly from the browser.
- Evidence of fix:
  - Settings no longer include `apiKey` (`src/types.ts:78-86`).
  - The settings UI no longer asks for a key and instead shows the backend route/handler (`src/overlay.ts:703-727`).
  - The runtime ignores deprecated `apiKey` input and warns instead of storing it (`src/overlay.ts:494-497`).

### Fixed-002: AI requests now send only non-secret field metadata
- Previous risk: Existing field values and scraped page context could be sent to providers.
- Evidence of fix:
  - AI request fields are reduced to metadata only via `GhostFillPromptField` and `toPromptFields()` (`src/types.ts:36-69`, `src/ai.ts:54-65`).
  - Prompt construction explicitly forbids relying on secrets or existing form values (`src/ai.ts:11-24`).
  - `describeFields()` no longer includes `currentValue` in the AI prompt (`src/detector.ts:176-197`).

## React / performance note

There is no React or Next.js runtime code in this repository to optimize directly. The relevant `react-best-practices` win applied here was bundle/runtime reduction: removing the unused page-context scraping path from `src/detector.ts` reduced client work and removed sensitive dead code from the shipped bundle.
